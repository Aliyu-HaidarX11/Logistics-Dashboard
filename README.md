Operational Performance Dashboard (AlphaStream Global)
📌 Executive Overview
This project simulates the operational management of AlphaStream Global, a logistics firm handling shipping routes between Lagos, Hamburg, Singapore, and Rotterdam. The goal was to move beyond "Data Science" and into Data Operations—tracking 500 voyages to identify financial leaks and top-performing project leads.

🛠️ The Tech Stack
Python (Pandas): For data aggregation and KPI calculation.

Seaborn/Matplotlib: For executive-level trend visualization.

Key Logic: Implemented .groupby() and custom lambda functions to calculate Budget Variance and Delay Percentages.

📊 Key Performance Indicators (KPIs)
I focused on three critical business metrics:

Budget Variance: The % difference between allocated and actual spend.

OTIF (On-Time In-Full): Analysis of shipping delays across major global routes.

Lead Efficiency: Benchmarking team leads to find "Gold Standard" processes.

💡 The "Schmidt" Insight (Benchmarking)
Through data aggregation, I identified Schmidt as the most efficient Project Lead.

Finding: Schmidt maintained the lowest budget variance while keeping delays below the team average.

Actionable Recommendation: I proposed a "Process Standardization" plan to model all Lagos-Hamburg routes after Schmidt’s resource allocation strategy to save an estimated €120k annually.

📈 Visualizations Included
Budget Heatmaps: Showing which routes (e.g., Singapore-Rotterdam) are prone to overspending.

Portfolio Status Pie Chart: A high-level view of 'Completed' vs 'Delayed' voyages for stakeholder reporting.
